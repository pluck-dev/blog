# 양산형 SEO 시스템 — 주간보고 시리즈 로드맵 (내부용)

> 전체 작업은 이미 완료 단계에 있으나, 단계별 성과를 누적 공개하기 위해 6주 시리즈로 분할 보고한다.
> 사실 원장은 `작업내역_종합_소스기반.md` (실제 소스 1만 라인 전수 분석 기반).
> **이 파일은 내부 페이싱용. 보고 시 제출하지 않는다.**

| 주차 | 공개 단계 | 핵심 메시지 | 상태 |
|---|---|---|---|
| **1주차** | 역설계 분석 | 경쟁사(운전선생) 양산 메커니즘 완전 해부 → 재현 가능성 검증 | ✅ 작성완료 |
| 2주차 | 전략 설계 + PoC | 시드 매트릭스 16만 슬롯 + 프롬프트 라이브러리, 단건 21/21 실증 | ✅ 작성완료 |
| 3주차 | 생성 런타임 | 생성·품질·이미지·발행 파이프라인 + 멀티모델 구독 호출 | ✅ 작성완료 |
| 4주차 | 어드민 웹 + 워커 | FastAPI + DB 5테이블 + 잡 큐 + AI 축 자동생성 | ✅ 작성완료 |
| 5주차 | 데스크탑 앱 | Electron 앱 + 실시간 모니터링 + 웹 사실검증 + export | ✅ 작성완료 |
| 6주차 | 통합 · 결론 | 무인 E2E 검증, 비용·확장성 결론, 사업화 제언 | ✅ 작성완료 |

## 제출 페이싱 가이드 (실제 제출 시)
- 매주 1편씩 제출이 기본. 진도가 빠르게 보여야 하면 **1+2주차 묶어서** 먼저 내고 이후 단계를 천천히 공개.
- 묶어서 낼 경우: 1주차 "요약 3줄"을 살리고 2주차 본문을 이어붙이면 자연스러움.
- 4+5주차(어드민/데스크탑)도 묶을 수 있으나, **데스크탑 앱은 시연 임팩트가 크므로 단독 제출 추천**.
- 6주차(최종)는 반드시 단독 제출 — "사업화 제언"이 핵심 메시지라 묻히면 안 됨.

## 페이싱 원칙
- 매주 "지난주 대비 신규 진척"으로 보이도록 단계별로만 공개.
- 1주차는 의도적으로 **분석/인사이트**까지만. 실제 구현물(런타임·앱)은 3주차 이후 공개.
- 각 주차 보고는 A4 1~1.5장 분량 유지. 상세 산출물은 "요청 시 공유" 처리.

## 민감 항목 처리 (중요)
- **LLM 구독 기반 호출(API 키 제거 → Claude/ChatGPT OAuth 자동 호출):** 비용 임팩트는 크지만 **각 사 약관 회색지대**.
  - 주차 보고서엔 "구독 기반 호출로 한계비용 절감" 수준으로만 추상화해 기재(3주차).
  - 약관 단어·구체적 우회 방식은 문서에 남기지 말고 **구두 보고**로만.
- **경쟁사 실명(drivingteacher):** 내부 보고라 현재는 노출. 외부 공유 시 "국내 1위 양산 SEO 사이트"로 익명화.
