## 1. 이번 주 핵심 요약 (3줄)

- 2주차의 1회성 PoC를 **상시 가동 가능한 자동 생성 런타임(약 2,000라인)으로 시스템화**했습니다.
- LLM 호출을 **두 개 모델(Claude·Codex)에 동시 대응**하도록 추상화하고, **구독 기반 호출로 호출 단가를 크게 낮추는 구조**를 적용했습니다.
- 품질 미달 콘텐츠를 **자동 재생성으로 끌어올리는 자가 교정 루프**와 **이미지 자동 수집·발행 HTML 생성**까지 한 흐름으로 연결했습니다.

---

## 2. 이번 주 진행 내용

### 2-1. 생성 런타임 모듈화 (7개 모듈, ~1,980라인)
콘텐츠 1건이 만들어지는 전 과정을 재사용 가능한 모듈로 분리했습니다.

| 모듈 | 역할 |
|---|---|
| **LLM 통합(llm)** | Claude·Codex 호출을 단일 인터페이스로 추상화, 응답 파싱·비용·토큰 집계 |
| **생성 오케스트레이터(generate)** | 프롬프트 렌더 → 호출 → 검증 → 재시도 → 저장 전 과정 지휘 |
| **품질검수(quality)** | 21개 항목 자동 검증 + 미달 시 보강 프롬프트 생성 |
| **이미지(images)** | 본문 이미지 슬롯을 Unsplash/Pexels에서 자동 수집·치환 |
| **프롬프트(prompts)** | 7종 템플릿(T01~T07) 빌더 |
| **발행(publish)** | 발행용 HTML + SEO 메타 + 구조화 데이터 생성 |
| **상태추적(slots)** | 슬롯별 생성 상태(대기/진행/완료/실패) 관리 |

### 2-2. LLM 호출 비용 최적화 (핵심)
- LLM 호출을 **종량제 API 키 방식이 아니라 구독 기반(CLI 인증)으로 동작**하도록 설계.
- → 토큰당 과금이 아닌 **월 구독료 범위 내에서 호출**하므로, 대량 생산 시 한계비용이 크게 낮아집니다.
- Claude·Codex **두 모델을 모두 지원**해, 한쪽 한도 소진 시 다른 모델로 전환 가능.
- *(주: 구독 기반 자동 호출은 각 사 약관 검토가 필요한 영역으로, 상세 운영 방식은 별도 구두 보고드리겠습니다.)*

### 2-3. 자가 교정 루프 + 후처리 자동화
- 생성 결과가 품질 기준(분량·구조·필수요소·금지표현)을 통과 못 하면 **미달 사항을 명시해 자동 재생성**(기본 2회).
- 통과 시 본문 이미지를 자동 수집하고, **발행 가능한 HTML(메타·CTA·구조화 데이터 포함)**까지 자동 생성.
- 모듈 간 느슨한 결합으로 이미지 수집 실패 등 일부 오류가 **전체를 중단시키지 않도록** 처리.

## 3. 의미 및 기대 효과

- **PoC → 양산 가능 시스템 전환:** "1건 됐다"에서 "계속 찍어낼 수 있다"로 단계 격상.
- **단가 추가 절감:** 구독 기반 호출로 대량 생산 시 LLM 비용 부담 대폭 완화.
- **품질 하한선 보장:** 자동 검수 + 자가 재생성으로, 사람이 일일이 보지 않아도 최소 품질 강제.
