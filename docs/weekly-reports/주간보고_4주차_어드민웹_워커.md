# 주간 업무 보고 — 4주차

**보고일:** 2026-06-05
**작성자:** 심재형
**주제:** 어드민 웹 + 백그라운드 워커 + 데이터베이스 구축

---

## 1. 이번 주 핵심 요약 (3줄)

- 3주차 런타임을 **웹에서 운영 가능한 관리 도구(FastAPI)와 자동 처리 워커**로 패키징했습니다.
- 고객사(테넌트)·축·슬롯·완성글·작업을 관리하는 **DB 5개 테이블 구조**를 설계하고, 작업을 큐에 쌓아 워커가 자동 소화하는 구조를 완성했습니다.
- **업종만 입력하면 LLM이 주제 축(지역·키워드·페르소나 등)을 자동 생성**하는 기능과 업종별 프리셋을 추가했습니다.

---

## 2. 이번 주 진행 내용

### 2-1. 어드민 웹 (FastAPI, ~1,570라인)
브라우저에서 양산 작업을 관리하는 콘솔을 구축했습니다.
- 대시보드(고객사 목록·현황), 고객사 추가/설정/삭제
- 축 관리(수동 입력 / 프리셋 / **AI 자동 생성**)
- 슬롯 자동 생성, 양산 작업 큐잉, 작업 모니터링, 완성글 미리보기
- 비밀번호 기반 접근 인증

### 2-2. 데이터베이스 설계 (SQLite, 5개 테이블)

| 테이블 | 관리 대상 |
|---|---|
| **tenants** | 고객사(도메인)별 설정 — 업종·테마·브랜드컬러·활성 템플릿·일일 한도 |
| **axes** | 주제 축 값 — 지역/키워드/인텐트/페르소나/수정자 + 검색량·경쟁도 |
| **slots** | 생성 후보(주제 조합) — 우선순위 점수·상태 |
| **posts** | 완성된 글 — 본문·메타·비용·토큰·생성정보 |
| **jobs** | 백그라운드 작업 — 종류·상태·결과 |

### 2-3. 백그라운드 워커
- 3초 간격 폴링으로 대기 작업을 가져와 처리 → **앱을 켜두면 작업이 알아서 소화**됨.
- 슬롯별로 생성→품질검증→재시도→발행/실패 기록, 작업 간 쿨다운으로 레이트리밋 방지.

### 2-4. AI 축 자동 생성 + 업종 프리셋
- **업종(예: 운전면허, ECU 맵핑)만 지정하면 LLM이 지역·키워드·인텐트·페르소나·수정자 5축을 자동 생성** → 신규 고객사 셋업 시간 대폭 단축.
- 운전면허·자동차 튜닝 등 **업종별 프리셋**을 미리 만들어, 클릭 한 번으로 주제 축 적용.

---

## 3. 의미 및 기대 효과

- **운영 자동화:** 작업을 큐에 넣으면 워커가 무인 처리 → 사람은 결과만 확인.
- **신규 고객사 온보딩 가속:** AI 축 생성·프리셋으로 "도메인 추가 → 주제 세팅"이 수 분 내.
- **멀티 고객사 기반:** 고객사별 데이터 분리 구조로 **여러 사이트 동시 운영(대행/SaaS) 토대** 확보.
