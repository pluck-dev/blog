## 1. 역설계 분석 (경쟁사 해부)

- 운전선생(`drivingteacher.co.kr`) 블로그 **21,275건 전수 스크래핑** (제목·본문·메타·발행시각·네이버 동시발행 여부).
- **분류 + 중복 검수 파이프라인** 구축 (MinHash/LSH, datasketch/sklearn 기반):
  - 본문 전체 중복률 **0.005%**, 같은 템플릿 내부 유니크율 **100%** 측정 → 목표 KPI로 채택.
- 노출/색인 전략 역설계 (사이트맵 블로그 제외, 색인 API 가속, 네이버 동시발행 백링크 등).
- 산출 문서: `PROGRAMMATIC_SEO_PLAYBOOK.md`, `PROMPT_LIBRARY.md`, `IMAGE_AND_TABLE_STRATEGY.md`, `COST_ANALYSIS.md`, `INDEX.md`.

## 2. 전략 설계 + PoC

- **시드 매트릭스**(`seed_matrix/`): 지역×키워드×인텐트×페르소나×수정자 5축 → **164,345 슬롯** 자동 도출 + 검색량·경쟁도 기반 우선순위 자동 채점.

## 3. 생성 런타임 — Python 파이프라인 (`runtime/`, 1,980 LOC)

### 3-1. LLM 통합 레이어 (`llm.py`, 399 LOC) — ★핵심
- **Claude / Codex 두 CLI를 동일 인터페이스로 추상화** (`run_llm(provider=...)`), 결과를 `LLMResult`로 정규화.
- **OAuth 구독제 강제 (= API 우회):**
  - Claude: `ANTHROPIC_API_KEY` / `ANTHROPIC_AUTH_TOKEN` 환경변수 **제거** → `claude --print - --output-format stream-json` 호출, claude.ai OAuth(Pro/Max) 자격증명 사용.
  - Codex: `OPENAI_API_KEY` / `CODEX_API_KEY` **제거** → `codex exec --json --sandbox read-only` 호출, ChatGPT OAuth(`~/.codex/auth.json`) 사용.
- **스트림 파싱:** Claude stream-json(assistant/result/rate_limit_event), Codex JSONL(thread.started/item.completed/turn.completed) 각각 파서 구현.
- **레이트리밋 추적:** Claude `rate_limit_event`의 7일 사용률 수집. 타임아웃 시 프로세스 kill, 바이너리 미설치 시 명확한 에러 안내.

### 3-2. 생성 오케스트레이터 (`generate.py`, 293 LOC)
- `generate_one()`: 프롬프트 렌더 → 상태 마킹 → LLM 호출 → 품질 검증 → **미달 시 보강 프롬프트로 재시도(기본 2회)** → md/json/images.json/html 저장.
- `run_batch()`: 슬롯 풀 순차 처리 + 슬롯 간 cooldown(레이트리밋 방지). argparse CLI(--provider/--slot/--csv/--templates/--min-priority/--limit/--cooldown/--model/--timeout/--with-images/--publish).

### 3-3. 품질 게이트 (`quality.py`, 111 LOC)
- `validate_post()`: H1 정확히 1개, H2 4~8개, 본문 2,300~3,600자, 이미지 슬롯 ≥3, 표 ≥1, 인용 ≥1, 불릿 ≥1, 내부링크 ≥1, "운전선생" 브랜드 포함, **AI 클리셰 표현 탐지**.
- `retry_prompt()`: 미달 항목을 명시한 재생성 프롬프트 생성 → 생성 루프와 연동.

### 3-4. 이미지 수집 (`images.py`, 281 LOC)
- `[IMAGE_SLOT: *]` 플레이스홀더를 **Unsplash/Pexels API**에서 검색·다운로드해 치환.
- **SHA1(slot_id:kind) 해시 기반 결정적 선택** → 양산글 간 이미지 분산 + 재생성 시 동일 이미지 유지.
- 한글 키워드→영문 매핑으로 스톡 검색 품질 향상, API 키 없으면 플레이스홀더 유지(안전 폴백).

### 3-5. 프롬프트 라이브러리 (`prompts.py`, 213 LOC)
- T01 지역 BEST5 / T02 단일업체 / T03 가이드 / T04 옵션비교 / T05 비용절약 / T06 시험팁 / T07 허브 — 7종 빌더 함수 + 공통 제약(분량·헤딩·슬롯·금지표현).

### 3-6. 발행 렌더러 (`publish.py`, 440 LOC)
- markdown→HTML 변환 + **운전선생 스타일 CSS 임베딩**(컬러 토큰/CTA 3종/지역검색 폼/푸터).
- 플레이스홀더 치환([IMAGE_SLOT]→figure 또는 박스, [INTERNAL_LINK]→a, [TABLE_SLOT]→제거), ⭐POINT 카드 자동 데코, **SEO 메타 head + JSON-LD(Article) 구조화 데이터** 생성.

### 3-7. 상태 추적 (`slots.py`, 176 LOC) / 호환 shim (`paperclip_runner.py`)
- CSV 슬롯 로더 + `output/state.json` 파일 기반 상태(planned/in_progress/published/failed) 추적, atomic write, DB 마이그레이션 고려 설계.

## 4. 어드민 웹 + 워커 — Python/FastAPI (`admin/`, 1,570 LOC)

### 4-1. 웹 서버 (`app.py`, 414 LOC)
- FastAPI + 쿠키 인증(`ADMIN_PASSWORD`). 주요 라우트:
  - 대시보드 `GET /`, 테넌트 CRUD(`/tenants`, `/t/{domain}`, settings, delete)
  - 축 관리(`/axes` 교체, `/axes/preset`, **`/axes/ai-fill` = LLM 자동 축 생성**)
  - 슬롯 생성(`/slots/generate`)·삭제·리셋
  - 양산 잡 큐잉(`/jobs`), 잡 모니터링(`/jobs`)
  - **워커 폴링 API**(`/api/jobs/claim`, `/api/jobs/{id}/complete`, `WORKER_TOKEN` 검증)
  - 글 미리보기(`/post/{id}` — 관리자 렌더 + 발행 미리보기)

### 4-2. DB 스키마 (`db.py`, 448 LOC) — SQLite, WAL, FK
- **tenants**(domain PK, vertical, theme, brand_color, templates_enabled, daily_limit…)
- **axes**(tenant, axis, value, weight, monthly_search_volume, competition_kd)
- **slots**(slot_id PK, template_id, primary_keyword, region/persona/intent/modifier_1·2, priority_score, status, last_error / INDEX tenant·status·priority)
- **posts**(id, slug, title, body_markdown, meta_description, provider, model, cost_usd, tokens, generated_at / UNIQUE tenant·slug)
- **jobs**(id, kind, payload, status, scheduled/started/finished_at, result / INDEX status·scheduled)

### 4-3. 백그라운드 워커 (`worker.py`, 216 LOC)
- 3초 폴링으로 `claim_next_job` → `generate` 처리: 슬롯별 프롬프트→LLM→품질검증→재시도(기본 2회)→발행/실패 기록 + cooldown. 결과 `{ok, fail, per_slot[]}`.

### 4-4. AI 축 생성 / 프리셋 / 슬롯 생성기
- `ai_axes.py`(149): 업종만 주면 LLM이 5축 값을 JSON으로 자동 생성 → DB 적재.
- `presets.py`(136): **driving / car-mapping / general** 업종 프리셋(축 값 사전 정의).
- `slot_gen.py`(175): 축 카르테시안 곱 + 우선순위 점수 `(sv_norm·0.6 + kd_norm·0.4)·weight·100`.

## 6. 기술 하이라이트 (보고 시 강조 포인트)

1. **LLM 비용 구조 혁신 (API 우회):** API 키 종량과금 대신 Claude/ChatGPT **정액 구독을 CLI로 자동 호출** → 한계비용을 구독료 수준으로 고정. *(주의: ToS 회색지대 — 보고 시 수위 판단 필요)*
2. **무인 품질 보증:** 21개 항목 자동 검증 + 미달 시 자가 재생성(최대 2회)으로 사람 검수 제거.
3. **할루시네이션 차단:** 웹 리서치로 실제 존재하는 상호/주소/가격만 사용하도록 검증 → 양산 콘텐츠의 신뢰성 확보.
4. **멀티테넌트 SaaS 기반:** 도메인별 데이터 분리 + 디자인 템플릿/브랜드 컬러 → 자체 운영과 고객 대행 양쪽 가능.
5. **이중 구현(웹/데스크탑):** 서버형 어드민 → 비개발자용 설치형 데스크탑 앱으로 진화, 실시간 진행 모니터링·4종 export까지.
